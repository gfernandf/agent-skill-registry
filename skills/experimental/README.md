# Experimental Skills

Skills in this channel are **self-assessed contributions** that have not yet
gone through peer review.

## Who can contribute

Any contributor.  No approval needed before publishing here — this is where
new ideas land first.

## Requirements

- Valid `skill.yaml` (passes `tools/validate_registry.py`)
- `metadata.status: experimental`
- At least items 1 (problem statement) and 4 (contract clarity) from the
  admission checklist answered in the PR description

## Stability guarantee

None.  Skills here may change signatures, be renamed, or be removed in any
release.

## Next step

Open a PR moving the skill to `community/` once peer review is sought.
See `docs/SKILL_ADMISSION_POLICY.md` for the full channel model.
